Cours GIT

Qu'est-ce que GIT

Git est outil de versionning, et il permet d'avoir une tracabilité et un historique de tous les fichiers du projet.

Il permet également de travailler en équipe : Envoyer nos modifications à nos collègues et récupérer les modifications des collègues.

Dans la tracabilité des fichiers GIT nous donne toujours 3 infos : Qui, Quand, Pourquoi ?

Les commandes de base

Initialiser un projet :
git init
Voir le statut des fichiers :
git status
Ajouter des fichiers :
git add 'fichier'
Ajouter TOUS LES FICHIERS :
git add .
Commit (enregistrer) les fichiers ajoutés :
git commit
Un commit nécessite toujours un message
commit avec message :
git commit -m "message"
commit avec add + message (fonctionne sur les fichiers déjà add précédemment) :
git commit -a -m "message"

Correction des fichiers

Voir les différents commit :
git log
Corriger les modifications sur un fichier (pas encore add) :
git checkout 'fichier'
Supprime un add :
git reset 'fichier'
git reset .
Supprimer le dernier commit :
git reset HEAD^
Supprimer les deux derniers commit :
git reset HEAD^^

La notion de branche

Une branche est une copie du projet. Elle permet de développer en parralèlle de nouvelles fonctionnalités. Lorsqu'on considère les nouveautés opérationnelles, on peut les rappatrier sur la version principale (master)

Voir les branches :
git branch
Créer une nouvelle branche :
git branch 'nom'
Se placer sur une branche :
git checkout 'nom'
Voir les différences entre deux branches :
git diff 'nom'..master
Apporter les modifs effectuées sur une branche dans master (il faut être sur master)
git checkout master
git merge 'nom'
Remplacer master par une branche
git checkout master
git fetch . 'nom'
Le fetch va littéralement remplacer master par la Branche. Il s'agit donc d'une grosse modification dans le projet (ex: version 2)

Repertoire distant et GitHub

On créé un nouveau repository sur GitHub, qui va faire office de serveur pour stocker le depot distant (qui lui sera à-même de "distribuer les fichiers aux collaborateurs du projet")

Pour connecter votre projet à votre depot distant :
Cette ligne est récupérable directement après la création du repository
git remote add origin https://github.com/identifiant/nom_depot_distant.git
Pour envoyer les fichiers au depot distant :
git push origin Master
A cette étape il faut être connecté à github. Votre password vous sera demandé

Pour travailler de manière collaborative :

Dans votre compte GITHUB vous devez ajouté des collaborateurs

Pour Récupérer les modifications des collègues :
git pull origin Master
Pousser ses modifs :
Apres avoir ADD et COMMIT bien entendu
git push origin Master
AVANT DE PUSH, on fait toujours un PULL. De telle sorte que : Si vous êtes le developpeur A, vous envoyez au Developpeur B, vos modifications, ainsi que celles que vous auriez récupérées du développeur C

Recap :

Récap des différentes etapes :

  1. J'arrive le matin... Je prend un café et je PULL, pour avoir une version à jour du projet
  2. je travaille durement toute la journée sur le projet...NO PAIN, NO GAIN :)
  3. La journée se termine, avant de partir, je :
    1. ADD mes fichiers créés ou modifiés
    2. COMMIT mes modifications
    3. PULL Les modifications des collègues
    4. PUSH mes modifications, ainsi que celles des collègues, pour le prochain
Notez toutefois que ces étapes peuvent être réalisées plusieurs fois par jour, ou tous les 2/3 jours selon votre rythme de travail, la taille de l'équipe... et plein d'autres critères qui sont propres à l'organisation de travail, et au volume du travail en cours.
